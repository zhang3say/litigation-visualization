# 诉讼可视化 (Litigation Visualization)

> 法律工作的本质是信息传递与处理。人类大脑对图形的理解天然优于文字。

遵循 [skills.sh](https://www.skills.sh/) 标准的 Agent Skill，将天同律师事务所《诉讼可视化》一书的方法论沉淀为 AI 可执行的画图工作流。帮助法律人将复杂的案件事实、法律关系转化为清晰图表，更好地说服法官、沟通客户。

## 核心方法论：四步工作法

```
明确对象 → 选择图表类型 → 确定内容 → 更好表达
```

### 第一步：明确对象

图表画给谁看决定了画什么、怎么画、甚至画不画。

| 对象 | 重点 |
|------|------|
| **法官** | 客观事实 + 己方观点，紧扣争议焦点 |
| **客户** | 诉讼策略和法律分析 |
| **内部团队** | 全面翔实，所有细节 |

### 第二步：选择图表类型

| 类型 | 适用场景 | 核心表达 |
|------|----------|----------|
| **时间图** | 时序是关键争议 | 时间轴 + 事件标注 |
| **关系图** | 主体众多、关系复杂 | 主体 + 线条连接 |
| **数据图** | 数据量大、需要对比 | 表格/柱状/饼状/折线 |
| **特殊图** | 地理位置、施工现场 | 路线图/平面图/立体图 |

### 第三步：确定内容

**3S 原则**：Simple（简单）+ Straight（直接）+ Strategy（策略）

1. **全面罗列** — 边看案卷边画细节图，事无遗漏
2. **逻辑整合** — 找到核心主体，按时间/关系主线组合
3. **精简内容** — 删除与争议焦点无关的信息

### 第四步：更好表达

**构图四原则**（源自 Robin Williams《写给大家看的设计书》）：
- **对比**：用颜色、字号、填充突出重点
- **重复**：统一配色、线宽、字体保持一致性
- **对齐**：用参考线，避免元素随意摆放
- **亲密性**：相关内容靠近，不同领域留白

**色彩规范**：
- 🔴 红色：警示、违约、不利事实
- 🔵 蓝色：正常履约、合规行为
- ⚫ 灰色：辅助信息、弱化内容
- 全案配色保持一致，不超过 3 种颜色，使用低饱和度
