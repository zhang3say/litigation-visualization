# 诉讼可视化

## 核心理念

法律工作的本质是信息传递与处理。人类大脑对图形的理解天然优于文字。诉讼可视化通过图表将繁复的案件材料转化为直观表达，提高信息传递效率。

## 四步工作法（核心流程）

每次绘制诉讼图表时，必须遵循以下四步：

```
明确对象 → 选择图表类型 → 确定内容 → 更好表达
```

---

### 第一步：明确对象 —— 图表画给谁看

| 呈送对象 | 图表重点 | 注意事项 |
|-----------|----------|----------|
| **法官** | 客观事实基础上突出己方观点，紧扣争议焦点 | 最重要的对象，最见律师功底 |
| **客户** | 展现对案件的把握、诉讼策略和法律分析 | 客户是亲历者，不需要重复事实 |
| **内部团队** | 内容翔实，增加事实细节，全面展示案情 | 用于讨论和分析 |

> **关键判断**：受众对某一领域是否了解，决定了是否需要画图。法官不需要再审流程图，客户不需要加油站建设流程图。

---

### 第二步：选择图表类型 —— 确定核心要素

案件的核心要素有三种，对应三种基本图表类型：

| 图表类型 | 适用场景 | 核心表达 |
|----------|----------|----------|
| **时间图** | 合同履行、程序流转、时序争议 | 时间轴 + 事件标注 |
| **关系图** | 股权结构、交易关系、主体众多 | 主体 + 线条连接 |
| **数据图** | 金额统计、比例对比、趋势变化 | 表格/柱状/饼状/折线 |

> 特殊场景：路线图（地理位置）、平面图（建设工程现场）、立体图（三维施工）、流程图（行政审批程序）。

#### 2.1 时间图（时间轴线图）

**何时使用**：核心争议在于事件发生的时间点、先后顺序或时间区间。

**基本结构**：中间画一条横线（或竖线），标注时间和事件。

**纵向空间利用**：
- **对比关系**：时间轴上方放一方行为，下方放另一方行为，形成对比
- **行为分层**：复杂案件可分子3-4层，每层放不同类别行为
  - 例：银行行为 / 信用社行为 / 基层法院行为 / 中院行为 / 高院行为

**横向时间轴加工**：
- **括号**：表达时间区间（保证期间、诉讼时效期间）
- **箭头**：表达持续的时间区间（如暂缓受理期间）
- **线条颜色变化**：区分不同阶段的状态

**时间图 checklist**：
- [ ] 确定核心时间节点（合同签订日、履行日、到期日等）
- [ ] 确定纵向空间分配（对比/分层）
- [ ] 确定是否需要表达「时间区间」（括号/箭头/颜色）
- [ ] 用两种颜色区分不同当事方行为
- [ ] 用红色标注违约/关键行为

#### 2.2 关系图（法律关系图）

**何时使用**：主体繁多（3个以上需梳理关系）；关系本身是争议焦点（融资性贸易、让与担保）；关系为叙事逻辑（股权结构变动）。

**绘制要领**：

1. **保持点线一致性**：主体用相同形状大小、线条含义一致（持股线全为实线，不中途变转让线）、同一形状不混用（切忌方圆三角混搭）
2. **主体尽量不重复**：合并同类项，同一主体只出现一次
3. **避免线条交叉、曲折**：通过调整主体排列顺序避免
4. **符合构图习惯**：股权结构股东在上子公司在下，箭头不代表两重含义

**常见结构**：
- **层级图**：股权/组织结构（股东上→子公司下）
- **循环图**：融资性贸易闭环（货物/资金流转画成环）
- **放射图**：以核心主体为中心向四周扩散

**关系图 checklist**：
- [ ] 确定所有参与主体
- [ ] 选择合适图形结构（层级/循环/放射）
- [ ] 统一主体图形形状和大小
- [ ] 确定线条含义（持股/资金流/合同关系/实际控制）
- [ ] 避免线条交叉
- [ ] 主体位置合理

#### 2.3 数据图

**何时使用**：数据量大繁杂、存在重要数据变动、需要强调数据间对比关系。

- **表格**：横纵向表达不同维度（横向=主体/时间，纵向=事件/变动）
- **柱状图**：数据间对比
- **饼状图**：比例关系
- **折线图**：趋势变化
- **混合使用**：折线图+事件标注 = 背景趋势+个案事实

---

### 第三步：确定内容 —— 筛选核心事实

**3S 原则**：

1. **Simple（简单）**：用最简文字，使用简称，不重复文字材料已有内容
2. **Straight（直接）**：事实描述要直接（不让读者自己归纳）；观点发表要直接（逻辑链完整无断点）
3. **Strategy（策略）**：精简有度，必须保留核心细节（具体条款、关键时间、关键语句）

**三步操作路径**：
1. **全面罗列**：分解案件为独立细节图，记下每个可作图的细节，事无遗漏
2. **逻辑整合**：找到核心主体作为连接点，按法律关系主线或时间主线组合
3. **精简内容**：删除与争议焦点无关的信息，只保留最有效的核心信息

---

### 第四步：更好表达 —— 设计实现

#### 4.1 构图四原则

源自 Robin Williams《写给大家看的设计书》：

- **对比**：不同元素差异明显（颜色、字重、填充），吸引注意力
- **重复**：统一颜色、线宽、字体、形状保持一致性
- **对齐**：元素有序排列，用参考线辅助
- **亲密性**：相关元素靠近，不同类别留出距离

#### 4.2 配色规则

| 颜色 | 含义 | 适用场景 |
|------|------|----------|
| 红色 | 强调、警示、违约 | 对方违约行为、争议焦点、不利事实 |
| 蓝色 | 正常、安全、履约 | 己方履约行为、合规程序 |
| 绿色 | 平静、安全、可行 | 已完成/已结算事项 |
| 灰色 | 模糊、辅助、弱化 | 无证据事实、非核心信息 |
| 黑色 | 中性、客观 | 基本框架、一般事实 |

**配色注意事项**：
- 全案图表保持一致的配色方案
- 有明确目的才使用颜色（不为好看而用）
- 颜色不超过3种为最佳
- 使用低饱和度颜色（法律工作的严肃性）
- 同一主体使用同一颜色
- 填充色与边框色尽量一致

#### 4.3 突出己方观点的技巧

**通过线条表达立场**：
- 实线 = 实际存在的法律关系/事实关系
- 虚线 = 名义上的/表层的/不存在的关系
- 粗线 = 强调重要性
- 箭头 = 资金流向、权利转移方向、控股方向

**通过颜色表达评价**：
- 同一颜色用于同一阵营主体
- 红色标注对方违约，蓝色标注己方履约

**通过框架表达阵营**：
- 虚线框架 = 利益共同体
- 实线框架 = 合同关系当事人
- 框架的划分直接体现律师对案件的分析结论

**其他技巧**：
- 避免线条交叉和长途奔袭
- 文字合理布局：详细说明放图表底部/侧面用脚注
- 图表不要过于拥挤，保持留白
- 同一案件不同立场：通过改变框架范围、虚实线使用、颜色分配来呈现截然不同的图表

## 各类型纠纷图表选择速查

| 纠纷类型 | 常见图表类型 | 绘图重点 |
|----------|-------------|----------|
| 借款合同 | 关系图、表格 | 签约主体 vs 实际履行主体、多笔借贷统计 |
| 票据纠纷 | 关系图+时间图+流程图 | 票面记载 vs 实际流转 vs 资金流向 |
| 建设工程 | 时间图、流程图、平面图 | 工期延误因果关系、竣工验收流程 |
| 土地开发 | 时间图、流程图、关系图 | 出让程序、合同履行顺序 |
| 国际货物买卖 | 流程图、关系图 | 贸易流程步骤、各方义务体系 |
| 公司纠纷 | 关系图（树状/层级） | 股权结构、股东身份、股权变动 |

## 典型案例绘图思路

### 案例1：买卖合同违约对比（时间图+对比布局）
双方各有违约，作为一方代理人：时间轴上方放己方履约（蓝色），下方放对方行为（红色=违约），读者自然对比关注。

### 案例2：多轮诉讼程序（时间图+行为分层）
银行与信用社多级法院诉讼：分5层（银行/信用社/基层/中院/高院），各层不同颜色，复杂程序立即清晰。

### 案例3：保证期间（时间图+括号表达区间）
两个司法解释计算方式不同：时间轴上标注关键节点，用括号标注两种计算方式下的保证期间范围，不同颜色区分。

### 案例4：公章使用（时间图+线条颜色区分阶段）
不同时段使用不同公章：时间轴不同阶段用不同颜色，对应标注该时期使用的公章样式。

### 案例5：融资性贸易（关系图+循环结构）
多家公司形成贸易闭环：主体按交易顺序排成环，标注同日签订的合同和价格差异，箭头表示货物流/资金流。

### 案例6：股权置换（关系图+上下对比）
股权置换前后对比：上图=置换前，下图=置换后，所有箭头表示静态持股，被转让资产标「×」。

### 案例7：减资纠纷（柱状图）
减资后又补足：两个等高柱子对比两次注册资本均为1亿元的时点，视觉直接传达「注册资本已恢复」。

### 案例8：多笔借款统计（表格）
数十笔借贷交叉：横列=金额/时间/利率/还款，纵列=各笔借款编号，借款/还款用不同颜色区分。

### 案例9：股权变动矩阵（横纵表格）
6次注册资本变动、10位股东：横向=股东，纵向=每次变动时间点，单元格=出资金额/变动。

### 案例10：心理动机展示（趋势图+事件叠加）
市场波动中的违约行为：上半部分=房地产指数折线图，下半部分=当事人关键行为，粗黑线=调控政策时间点。

### 案例11：票据纠纷（关系图+三线合一）
票据虚假背书贴现：蓝色=票面记载流转，黑色=实际流转，红色=资金流向，三线合一揭示实质借贷关系。

## 制图实战流程

1. **阅卷→全面罗列**：边看案卷边画细节图，每个可画图的法律关系都画
2. **找核心主体→逻辑整合**：找到反复出现的核心主体，以它为连接点组合各细节图
3. **分析争议焦点→精简内容**：删除与核心争议无关的信息，保留关键细节
4. **确定立场→设计表达**：用颜色、虚实线、框架突出己方观点
5. **迭代优化**：不断试错，不断简化，追求「极简表达」

## 常见误区

- ❌ 把案件大事记内容直接塞进方框 → 没有信息加工
- ❌ 箭头同时表示两种关系（如既表示持股又表示转让）→ 读者混淆
- ❌ 同一张图用不同形状（方圆三角混搭）→ 不专业
- ❌ 主体在图中重复出现 → 结构混乱
- ❌ 线条交叉、长距离奔袭 → 影响阅读
- ❌ 颜色超过3种 → 花哨，失去法律严肃感
- ❌ 文字表述模糊（如「相关约定」）→ 失去信息价值
- ❌ 信息过于拥挤不留白 → 压迫感
- ❌ 对法官呈送纯观点关系图而非事实图 → 缺乏说服力
